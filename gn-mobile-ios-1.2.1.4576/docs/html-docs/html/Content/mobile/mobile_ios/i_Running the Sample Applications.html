Building and Running the Sample Applications

To run either of the sample applications:

  1. Unpack the Gracenote iOS GNSDK for Mobile package: gn-mobile-ios-<version>.zip
  2. Copy your license file, names as "license.txt", into the package root directory. The Xcode project expects the license file to have this name and be in this location. To use a different license file name and location you can update the project.
  3. Launch Xcode and open the project <package root>/mmid/GracenoteMusicId.

    This project contains two apps that share some resources like the GNSDK Objective-C framework. Both are GNSDK sample apps, one is an Objective-C app, the other a Swift app.

To run the Objective-C app:

  1. Select target GracenoteMusicId
  2. Set your Client IDEach customer receives a unique Client ID string from Gracenote. This string uniquely identifies each application and lets Gracenote deliver the specific features for which the application is licensed. The Client ID string has the following format: 123456-789123456789012312. The first part is a six-digit Client ID, and the second part is a 17-digit Client ID Tag. and Tag in the prefix pre-compiled header located at MusicIdObjectiveC/MusicidObjectiveC-Prefix.pch
  3. Set the View Controller for Main_iPhone storyboard to GnViewController
  4. Clean, build, and run

To run the Swift app:

  1. Select target GracenoteMusicIdSwift
  2. Set your Client ID and Tag in GnViewControllerSwift.swift
  3. Set the View Controller for Main_iPhone.storyboard to GnViewControllerSwift
  4. Clean, build, and run
